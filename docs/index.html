Standalone Python validator — no AI needed. Runs in CI, checks schema + governance rules, strict mode on main branch.

Reads manifest, resolves capability packs, materialises agent + standard files into .claude/. Dry-run safe, idempotent.

eeik-validate.yml

CI Workflow
eeik-adapt.yml
On manifest change to main — auto-generates all 6 adapter files, commits with [skip ci]. Manual dispatch with dry_run and tools inputs.

Philosophy

The Vision

Projects declare what they need. EEIK determines how it should be built.

❌ The Problem Today

  • 🔁 Every project recreates the same engineering standards
  • 🧱 Knowledge lives in silos, never reused
  • ⏳ Project setup takes weeks, not hours
  • 🚨 Governance added late — after architecture drift
  • 🤷 AI assistants have no enterprise context
  • 📉 Repeated mistakes no one recorded

✅ EEIK's Answer

  • ♻️ Capability packs: build once, reuse everywhere
  • 📚 Org knowledge baked into every session
  • ⚡ Bootstrap to working repo in <30 minutes
  • 🛡️ Governance as code — from manifest, day one
  • 🤖 44 specialist agents × 6 AI tools, context-aware from the start
  • 🧬 Every project enriches the platform

Getting Started

Adopting EEIK — 6 AI Tools, One Bootstrap

Under 30 minutes for any project. Run the bootstrap script to generate all 6 AI tool adapters from your manifest, then activate the capability packs your project needs.

Claude Code GitHub Copilot Kiro Codex CLI Cursor Gemini CLI
1

Clone & Copy Config Layers

Fork or clone eeik_bootstrap, then copy config layers into your new project directory.

EEIK=/path/to/eeik_bootstrap mkdir my-new-service && cd my-new-service git init cp -r $EEIK/.claude ./.claude cp -r $EEIK/.github ./.github cp -r $EEIK/.kiro ./.kiro cp -r $EEIK/.cursor ./.cursor cp $EEIK/AGENTS.md ./AGENTS.md cp $EEIK/GEMINI.md ./GEMINI.md # ⚠️ Use templates/PROJECT-CLAUDE.md — NOT EEIK's own CLAUDE.md # EEIK's CLAUDE.md describes the EEIK repo structure. If you copy it, # Claude Code thinks it's inside EEIK and writes artifacts there. cp $EEIK/templates/PROJECT-CLAUDE.md ./CLAUDE.md
2

Validate the Manifest

Run the standalone validator (no AI needed) to catch schema errors and governance rule violations before generating anything.

python3 scripts/validate_manifest.py project-manifest.yaml # → passes 8 governance rules (G001–G004, T001–T003, P001–P002) # → --strict flag for CI gates on main branch # Or in Claude Code: /validate-manifest
3

Generate All 6 AI Tool Adapters

One script generates adapters for every tool you enabled in the manifest. Each adapter reads from the same shared capability packs — intelligence defined once, delivered everywhere.
